# Желание — OneClickCharity

Прототип благотворительного сервиса, где даритель видит **конкретного ребёнка, конкретный подарок и результат доставки**. В отличие от привычных пожертвований в «общий фонд», здесь закрывается одно желание за раз — ребёнку нужен футбольный мяч, набор для рисования или зимняя куртка, и даритель оплачивает именно этот предмет.

Это **статический HTML/CSS/JS-макет без бэкенда**: весь каталог и данные живут в памяти, модерация и доставка симулируются `setTimeout`-ами. Назначение — показать end-to-end пользовательский опыт и передать UX-логику команде, которая будет делать production-версию.

## Запуск

Открыть `wishful_app_2.html` напрямую в браузере, либо поднять статический сервер из корня репо:

```bash
python3 -m http.server 8000
```

и зайти на <http://localhost:8000/wishful_app_2.html>.

Никакой сборки, npm или тестового раннера не требуется.

## Экраны (8 шт.)

1. **Welcome** — выбор роли: даритель или виш-мейкер.
2. **Wish creation** — форма желания (имя, возраст, город, категория, бюджет, благодарность).
3. **Moderation** — симулированная проверка заявки и подбор товара.
4. **Donor catalog** — список желаний с поиском и фильтрами по бюджету/благодарности.
5. **Wish details** — карточка ребёнка + подходящий товар + блок прозрачности.
6. **Payment** — выбор способа оплаты, анонимность, сообщение команде.
7. **Tracking** — таймлайн «оплата → выкуп → доставка → вручение».
8. **Result** — итог с благодарностью и голосовым сообщением (модалка).

## Архитектура кратко

- **Единый `state`-объект** в `app.js` хранит выбранную роль, желание, бюджет, анонимность, шаг трекинга.
- **Экраны** — соседние `<div class="screen">`, активен один за раз; переходы через `goTo(id)`. Роутера нет.
- **Рендер** — template-strings + `innerHTML` в функциях `renderCatalog`, `renderDetails`, `renderPaymentScreen`, `renderTracking`, `renderResult`.
- **Пользовательский ввод** (`name`, `city`, `wish`, `reason`) пропускается через `escapeHtml()` перед вставкой в шаблоны — это обязательно.
- **Данные** — массив `demoWishes`. Созданные желания кладутся в начало `catalogItems` (сбрасываются при перезагрузке).
